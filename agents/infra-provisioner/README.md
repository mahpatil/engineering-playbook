# Infra Provisioner Agent

Generates production-grade Terraform modules for AWS, GCP, and Azure based on the organisation's infrastructure standards. Enforces mandatory tagging, zero-trust networking, remote state, and DR requirements automatically.

## Prerequisites
- This repo cloned locally or accessible via API.
- `standards/claude-md/infra/CLAUDE.md` present (merged from the standards PR).
- Claude Code CLI (`claude`) installed, or access to the Anthropic API.

---

## Invocation Methods

### 1. Claude Code (interactive)
```bash
# From repo root
claude --agent agents/infra-provisioner/AGENT.md

# Then paste your request, e.g.:
# "Provision a prod-tier AWS environment for payments-api with DR tier 1"
```

### 2. Claude Code with `--print` (scripted / CI)
```bash
claude --agent agents/infra-provisioner/AGENT.md \
  --print \
  --input agents/infra-provisioner/example-input.md \
  > infra/payments-api/prod/generated.md
```

### 3. Anthropic API (Python)
```python
import anthropic
from pathlib import Path

client = anthropic.Anthropic(api_key="YOUR_API_KEY_HERE")

agent_prompt = Path("agents/infra-provisioner/AGENT.md").read_text()
user_request = Path("agents/infra-provisioner/example-input.md").read_text()

response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=8096,
    system=agent_prompt,
    messages=[{"role": "user", "content": user_request}],
)
print(response.content[0].text)
```

### 4. GitHub Actions (CI)
```yaml
# .github/workflows/infra-provision.yml
name: Infra Provision

on:
  workflow_dispatch:
    inputs:
      app_name:
        required: true
      cloud_target:
        required: true
        default: aws
      environment:
        required: true
        default: dev
      dr_tier:
        required: true
        default: "3"

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Generate Terraform modules
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          python scripts/run_agent.py \
            --agent agents/infra-provisioner/AGENT.md \
            --app-name "${{ inputs.app_name }}" \
            --cloud-target "${{ inputs.cloud_target }}" \
            --environment "${{ inputs.environment }}" \
            --dr-tier "${{ inputs.dr_tier }}" \
            --output-dir "infra/${{ inputs.app_name }}/${{ inputs.environment }}"

      - name: Open PR with generated Terraform
        run: gh pr create --title "infra: provision ${{ inputs.app_name }} ${{ inputs.environment }}" --body "Auto-generated by infra-provisioner agent"
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Inputs

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `app_name` | string | Yes | Lowercase, hyphen-separated app identifier |
| `cloud_target` | enum | Yes | `aws`, `gcp`, or `azure` |
| `environment` | enum | Yes | `dev`, `staging`, or `prod` |
| `dr_tier` | 0–3 | Yes | 0=no DR → 1=RPO<1h active-passive |
| `region` | string | No | Primary region (default per cloud) |
| `secondary_region` | string | Cond. | Required if `dr_tier` is 0 or 1 |
| `team` | string | No | Owning team name (tag) |
| `cost_centre` | string | No | FinOps cost centre code (tag) |

---

## Outputs

The agent produces a directory `infra/<app_name>/<environment>/` containing:

| File | Purpose |
|------|---------|
| `main.tf` | Provider, resources, security groups, IAM, encryption |
| `variables.tf` | Typed, described variable declarations |
| `outputs.tf` | Resource IDs, endpoints, ARNs |
| `backend.tf` | Remote state backend config |
| `terraform.tfvars.example` | Safe placeholder example vars file |

Plus a **Provisioning Checklist** at the end of the response.

---

## Pipeline Integration

This agent is Step 1 of the provisioning → deployment pipeline:

```
infra-provisioner  →  (review & apply Terraform)  →  app-deployer
```

Once Terraform is applied and cluster/compute exists, pass the outputs (cluster endpoint, registry URL, namespace) to the `app-deployer` agent.

---

## Extending for Your Org

1. Add your org's tagging taxonomy to `standards/claude-md/infra/CLAUDE.md`.
2. Update the `tags` block in `AGENT.md` to add new mandatory tags.
3. Add cloud-specific modules (e.g. Transit Gateway, Shared VPC) as additional output files.
